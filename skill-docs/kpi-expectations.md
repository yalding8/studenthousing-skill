# uhomes Skill — KPI 预期框架

| Skill 发现机制 | 以 ClawHub 向量搜索为主，用户主动安装；尚无应用内推荐算法 |

### 趋势研判

OpenClaw 生态在 2026 年下半年有两个值得关注的变量：

一是随着 AI Agent 工具向非技术用户渗透，OpenClaw 或其分叉/继承项目的用户结构会逐渐多元化。具备生活场景（住宿、出行、学习）的 Skill 会随之获得更多曝光。

二是 MCP（Model Context Protocol）作为跨平台标准正在收敛，兼容 MCP 的 Skill 可以同时被 Claude.ai、OpenClaw、以及未来更多 Agent 平台发现和调用，分发边界将超出 ClawHub 本身。

**结论**：uhomes Skill 现在发布，是以较低成本占据这一品类的先发位置。价值在于"已在那里"，而非立即产生可观的转化流量。

## 2. KPI 框架

### 2.1 指标分层

```
第一层（必跟踪）：可见性指标
  ↓ 若可见性达标
第二层（核心评估）：参与质量指标
  ↓ 若参与质量达标
第三层（业务验证）：转化归因指标
```

### 2.2 具体指标与分阶段目标

#### 第一层：可见性指标

| 指标 | 定义 | 发布后1个月 | 发布后3个月 | 发布后6个月 |
|------|------|-------------|-------------|-------------|
| ClawHub 安装量 | 累计 install 数 | 100 | 500 | 2,000 |
| ClawHub 评分 | 用户评分均值 | N/A（样本太小） | ≥ 4.0 | ≥ 4.2 |
| 描述触发率 | 用户搜索住宿时 Skill 被调用比例（人工抽样测试） | ≥ 75% | ≥ 80% | ≥ 85% |

> 安装量基准参考：ClawHub 头部 Skill 安装量在数千至数万量级，但多为通用工具类（web search、calendar 等）。垂直类 Skill 首年达到 1,000-5,000 安装属于健康区间。

#### 第二层：参与质量指标

| 指标 | 定义 | 发布后1个月 | 发布后3个月 | 发布后6个月 |
|------|------|-------------|-------------|-------------|
| UTM 会话数/月 | 通过 Skill UTM 链接到达 uhomes.com 的 GA4 会话 | 50 | 300 | 1,000 |
| 深链点击率 | 用户收到 Skill 推荐后点击 uhomes 链接的比例 | — | ≥ 35% | ≥ 45% |
| 页面停留时间 | Skill 渠道用户在 uhomes.com 的平均会话时长 vs. 整体均值 | — | 差距 ≤ -20%（可接受） | 与整体均值持平 |

> 页面停留时间初期可能低于均值，因为 Skill 用户已被预筛选了意图，直接进入目标页后行为更集中。如果显著低于均值（> -30%），说明推荐质量有问题。

#### 第三层：转化归因指标

| 指标 | 定义 | 发布后3个月 | 发布后6个月 |
|------|------|-------------|-------------|
| 归因预订意向数/月 | `booking_initiated` 事件中来自 Skill UTM 的次数 | 5 | 20 |
| 归因预订完成数/月 | 实际完成预订（如能追踪）| 1 | 5 |

> **重要说明**：转化归因指标受限于跨设备行为（用户在 OpenClaw 看到推荐、在手机完成预订），实际贡献可能被低估 30-50%。以 `booking_initiated` 而非"完成预订"作为主要评估指标更合理。

## 3. 分阶段评估节点

### 第一次评估（发布后 4 周）
**评估问题**：Skill 能被正常发现和使用吗？

通过标准：
- ClawHub 安装量 ≥ 50
- 描述触发率 ≥ 75%（人工测试 20 个场景）
- UTM 数据在 GA4 中正常出现（可见性验证）

**失败处理**：调整 description 措辞，提交 ClawHub 重新审核；检查 UTM 配置。

### 第二次评估（发布后 3 个月）
**评估问题**：渠道质量是否达到继续投入的阈值？

通过标准：
- UTM 月会话 ≥ 300
- 深链点击率 ≥ 35%
- 安装量增速持续为正

**决策路口**：
- 如果质量指标达标 → 投入资源扩展城市覆盖（v1.1），考虑开发 API 接口替换 web_fetch
- 如果质量指标未达标但安装量正常 → 优化对话流程和推荐质量，不急于扩展
- 如果安装量和质量均未达标 → 评估是否值得在 Claude.ai 侧独立推广此 Skill

### 第三次评估（发布后 6 个月）
**评估问题**：该渠道的长期投入产出比是否合理？

通过标准（任意一项）：
- 月归因预订意向 ≥ 20（显示实质转化潜力）
- 安装量达到 2,000+（生态地位稳固，未来随平台增长被动受益）
- MCP 标准扩散带来其他平台的 Skill 调用量增长

## 4. 不应使用的对比基准

以下基准不适用于 Skill 渠道，避免用其考核导致错误决策：

| 不适用基准 | 原因 |
|------------|------|
| SEO 自然流量的转化率（通常 2-5%） | Skill 用户行为模式与搜索意图用户不同，且样本量早期极小 |
| 付费广告的 CPA（Cost per Acquisition） | Skill 无直接投放成本，成本结构完全不同 |
| 微信生态的流量指标 | 用户群体重叠度低，不具参考价值 |
| 同类 uhomes 渠道的首月 GMV 目标 | Skill 是品牌基础设施，不是即时 GMV 渠道 |

## 5. 长期价值维度（非 KPI，但值得记录）

以下是难以量化但具有真实战略价值的收益，供长期决策参考：

**品类先发**：uhomes 是目前 ClawHub 上唯一的留学生住宿搜索 Skill。这个位置在品类关键词搜索中的先发优势，随平台用户增长会被动放大。

**MCP 生态布局**：如果 uhomes Skill 兼容 MCP 标准，未来可以被 Claude.ai、Cursor、其他支持 MCP 的工具直接调用，分发边界远超 OpenClaw 本身。

**用户画像洞察**：Skill 的使用数据（哪些城市被问最多、哪些条件被筛选最频繁）可以反哺 uhomes 的内容和房源运营策略，这是其他渠道难以提供的结构化信号。
